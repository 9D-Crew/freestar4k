# Contributing to FreeStar 4000

This is a WIP guide on what to do (or not to do) when you want to contribute.

## What to do before making an issue

* Double check to see if it's user error

## What to do when making an issue

* Videos are preferred for showcasing specific issues
* For broader issues, a text description may suffice

## What to do before making a pull request

* If adding a feature, determine if it would be more fitting to create an extension.

## What to do when making a pull request

* Don't bump the version number or unstable letter
* Please make the purpose clear
